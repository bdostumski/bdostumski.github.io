#+title: bdostumski.github.io
#+author: Borislav Dostumski

Personal website and blog of [[https://github.com/bdostumski][Borislav Dostumski]], built entirely with
*GNU Emacs*, *Org Mode*, and published to *GitHub Pages*.

Live at: https://bdostumski.github.io

-----

* Stack

| Layer      | Technology                                     |
|------------+------------------------------------------------|
| Content    | Org Mode (~.org~ files in ~org/~)              |
| Build      | ~ox-publish~ + ~htmlize~ (via ~build-site.el~) |
| Styling    | Hand-written CSS — Doom Emacs ~doom-one~ theme |
| JavaScript | Vanilla JS — modeline, mobile nav, code copy   |
| Hosting    | GitHub Pages                                   |
| CI / CD    | GitHub Actions (~.github/workflows/main.yml~)  |

No static site generator. No Node.js. No build framework.
Just Emacs publishing Org files to HTML.

* How It Works

** Building locally

Requires *Emacs 29.1+*.

#+begin_src sh
./build.sh
#+end_src

This runs:

#+begin_src sh
emacs -Q --script build-site.el
#+end_src

The ~-Q~ flag starts Emacs without any personal config (no ~init.el~, no
Doom, no packages), keeping the build clean and reproducible.

~build-site.el~ then:

1. Bootstraps the package system, pointing ~package-user-dir~ to a local
   ~.packages/~ directory so nothing is installed globally.
2. Fetches package archives (MELPA + GNU ELPA) if not already cached.
3. Installs ~htmlize~ — used to syntax-highlight source code blocks in HTML.
4. Loads ~ox-publish~ and configures two publish components:
   - ~site-org~ — converts every ~.org~ file under ~org/~ to HTML in ~public/~
   - ~site-static~ — copies CSS, JS, and image assets from ~static/~ to ~public/static/~
5. Runs ~(org-publish-all t)~ with ~org-publish-use-timestamps-flag nil~
   so every file is always rebuilt.
6. Org Babel blocks (Emacs Lisp) inside ~blog.org~, ~certificates.org~, and
   ~index.org~ execute at publish time to auto-generate post and certificate
   lists from the filesystem — no manual index maintenance needed.

Output lands in ~public/~.

** Preview locally

After building, open Emacs and run:

#+begin_example
M-x httpd-serve-directory
#+end_example

Point it to the ~public/~ directory when prompted.
Then open http://localhost:8080 in your browser.

To stop the server: ~M-x httpd-stop~.

** Deployment

Deployment is fully automatic via GitHub Actions.

On every push to ~main~, the workflow at ~.github/workflows/main.yml~:

1. Checks out the repository.
2. Configures GitHub Pages.
3. Uploads the ~public/~ directory as the Pages artifact.
4. Deploys it to https://bdostumski.github.io.

*Note:* the build step does not run in CI — ~public/~ is committed directly
to the repository and served as-is. Run ~./build.sh~ locally and commit the
result.
