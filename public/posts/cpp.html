C++ Programming Q&A (Beginner → Expert)

Beginner Level

Q1: What is C++?

C++ is a general-purpose programming language that supports procedural, object-oriented, and generic programming, with direct hardware control and high performance.

Q2: What does a minimal C++ program look like?

#include <iostream>

int main() {
    std::cout << "Hello, world!\n";
    return 0;
}

Q3: What is the difference between C and C++?

C++ adds classes, templates, RAII, exceptions, overloading, namespaces, and richer standard libraries on top of C-like low-level capabilities.

Q4: What is a namespace?

A namespace groups identifiers to avoid name collisions. 

namespace math { int add(int a, int b) { return a + b; } }

Q5: Why avoid using namespace std; in headers?

It pollutes global namespace and can cause symbol conflicts for users of the header.

Q6: What is std::cout?

Standard output stream object from <iostream>.

Q7: What is std::cin?

Standard input stream object for formatted input.

Q8: What is a variable declaration?

Defining a named object with a type, e.g., int x = 5;.

Q9: What are common built-in types in C++?

bool, char, int, float, double, void plus signed/unsigned and width variants.

Q10: What is auto in C++?

Type deduction keyword where compiler infers type from initializer. 

auto x = 42;      // int
auto y = 3.14;    // double

Q11: What is the difference between = and == ?

  • = assigns
  • == compares equality

Q12: What are references?

Aliases to existing objects, typically safer than raw pointers for mandatory binding. 

int x = 10;
int& r = x;

Q13: Difference between pointer and reference?

  • Pointer can be null/reseated
  • Reference must bind to valid object and cannot be reseated after initialization

Q14: What is const?

Qualifier preventing modification through that name/reference/pointer view.

Q15: What is const int* vs int* const?

  • const int* p: pointer to const int
  • int* const p: const pointer to int

Q16: What is a function prototype?

Declaration of function signature before use.

Q17: What is function overloading?

Same function name with different parameter lists. 

int f(int);
double f(double);

Q18: What is default argument?

Parameter default value used if caller omits argument. 

void log(int level = 1);

Q19: What is an inline function?

Suggests inlining and allows multiple identical definitions across translation units.

Q20: What is a class?

User-defined type bundling data and behavior with access control.

Q21: Difference between class and struct in C++?

Default access is private for class, public for struct.

Q22: What are access specifiers?

public, private, protected control member visibility.

Q23: What is encapsulation?

Hiding implementation details and exposing stable public interface.

Q24: What is a constructor?

Special member that initializes objects.

Q25: What is a destructor?

Special member called when object lifetime ends, for cleanup.

Q26: Can constructors be overloaded?

Yes, multiple constructors with different parameter lists are common.

Q27: What is initializer list in constructor?

Preferred member initialization syntax before constructor body. 

struct S {
  int x;
  S(int v) : x(v) {}
};

Q28: Why prefer initialization over assignment in constructor body?

Direct initialization avoids extra work and is required for const/reference members.

Q29: What is RAII?

Resource Acquisition Is Initialization: bind resource lifetime to object lifetime.

Q30: Why is RAII important?

Prevents leaks and ensures exception-safe cleanup automatically.

Q31: What is new and delete?

Dynamic allocation/deallocation operators; prefer smart pointers/containers in modern C++.

Q32: Why avoid raw new/delete in application code?

Error-prone ownership management; use std::unique_ptr, std::vector, etc.

Q33: What is nullptr?

Type-safe null pointer literal introduced in C++11.

Q34: Difference between NULL and nullptr?

nullptr has dedicated null pointer type; avoids overload ambiguity.

Q35: What is an array?

Fixed-size contiguous collection of same type.

Q36: Why prefer std::array over raw array for fixed size?

Value semantics, size awareness, iterators, STL compatibility.

Q37: Why prefer std::vector for dynamic arrays?

Automatic memory management, resizing, contiguous storage, rich API.

Q38: What is a string in C++?

std::string is dynamic, owning text container; easier/safer than C strings.

Q39: What is range-based for loop?

for (const auto& v : vec) { /* ... */ }

Q40: What is type inference benefit?

Reduces verbosity and keeps declarations aligned with initializer types.

Intermediate Level

Q41: What is copy constructor?

Constructor initializing object from another same-type object.

Q42: What is copy assignment operator?

Assigns one existing object from another same-type object.

Q43: What is move constructor?

Transfers resources from temporary/rvalue object.

Q44: What is move assignment operator?

Transfers resources into already existing object from rvalue.

Q45: What is Rule of Three?

If class defines destructor/copy ctor/copy assign, likely needs all three.

Q46: What is Rule of Five?

Rule of Three extended with move ctor and move assign.

Q47: What is Rule of Zero?

Prefer designing classes needing no custom special members (delegate ownership to RAII members).

Q48: What is = default?

Ask compiler to generate default implementation of special member.

Q49: What is = delete?

Explicitly forbid a function/special member. 

MyType(const MyType&) = delete;

Q50: What is lvalue vs rvalue?

Lvalue has identity/location; rvalue is temporary/value-like expression.

Q51: What is std::move?

Cast to rvalue enabling move semantics; does not move by itself.

Q52: What is moved-from state?

Valid but unspecified state after move unless otherwise documented.

Q53: What is perfect forwarding?

Preserving value category through templates via forwarding references + std::forward.

Q54: What is forwarding reference?

T&& in deduced template context, can bind lvalue/rvalue.

Q55: What are exceptions?

Error handling mechanism with throw, try, catch.

Q56: When avoid exceptions?

In low-latency/hard real-time/no-exception codebases; depends on project policy.

Q57: What is stack unwinding?

Destructor calls for in-scope objects when exception propagates.

Q58: Why destructors should not throw?

Throwing during unwinding can terminate program.

Q59: What does noexcept mean?

Function guarantees not to throw (or program terminates on violation).

Q60: Why mark move operations noexcept?

Enables containers to use move during reallocation for performance/safety.

Q61: What is template?

Compile-time generic blueprint for functions/classes.

Q62: What is function template?

template<typename T>
T add(T a, T b) { return a + b; }

Q63: What is class template?

Generic class parameterized by types/values.

Q64: What is template specialization?

Custom implementation for specific template arguments.

Q65: What is partial specialization?

Specialized template for subset/pattern of parameters (class templates only).

Q66: What are non-type template parameters?

Template parameters that are values (e.g., int N, pointer, auto in newer standards).

Q67: What is constexpr?

Enables compile-time evaluation when possible.

Q68: Difference between const and constexpr?

  • const: immutable object
  • constexpr: constant expression capable (if context permits)

Q69: What is if constexpr?

Compile-time conditional branch in templates.

Q70: What is staticassert?

Compile-time assertion with diagnostic message.

Q71: What is STL?

Standard Template Library components: containers, iterators, algorithms, functors.

Q72: Common STL containers?

vector, deque, list, map, unordered_map, set, unordered_set, array, string.

Q73: How choose map vs unordered_map?

  • map: ordered, tree, O(log n)
  • unordered_map: hash table, average O(1), no ordering

Q74: What are iterators?

Generalized pointers for traversing container elements.

Q75: Difference between begin/end and cbegin/cend?

cbegin/cend return const iterators.

Q76: What are algorithms in <algorithm>?

Reusable generic operations like sort, find, transform, copy.

Q77: Why prefer algorithms over handwritten loops?

Clarity, correctness, composability, and often optimized implementations.

Q78: What is lambda expression?

Anonymous function object. 

auto sq = [](int x){ return x * x; };

Q79: What is lambda capture?

How lambda accesses outer variables ([=], [&], explicit captures).

Q80: What is mutable lambda?

Allows modification of by-value captures inside lambda body.

Q81: What is std::function?

Type-erased callable wrapper for functions/lambdas/bind targets.

Q82: Cost of std::function?

Type erasure may allocate and add indirection; templates/auto may be cheaper in hot paths.

Q83: What is std::bind?

Binds callable arguments; lambdas are usually clearer modern alternative.

Q84: What is enum class?

Scoped, strongly typed enum preventing implicit integer conversions.

Q85: Why use enum class over enum?

Better type safety and namespace hygiene.

Q86: What is friend in C++?

Grants selected functions/classes access to private/protected members.

Q87: What is operator overloading?

Defining behavior of operators for user-defined types.

Q88: Which operators should be overloaded carefully?

&&, ||, comma, &, conversion operators—can surprise readers.

Q89: Why overload << for printing?

Integrates type with stream output idioms. 

std::ostream& operator<<(std::ostream& os, const T& t);

Q90: What is explicit constructor?

Prevents unintended implicit conversions.

Q91: What is conversion operator?

Member enabling conversion to another type (prefer explicit where possible).

Q92: What is object slicing?

Copying derived object into base object by value, losing derived parts.

Q93: How avoid slicing?

Use references/pointers (often smart pointers) to polymorphic base.

Q94: What is inheritance?

Deriving new class from existing base class.

Q95: What is polymorphism?

Same interface, different behavior by concrete dynamic type.

Q96: What is virtual function?

Function dispatched at runtime based on dynamic type.

Q97: Why virtual destructor in polymorphic base?

Ensures correct derived cleanup when deleting via base pointer.

Q98: What does override mean?

Compile-time check that function overrides base virtual. 

void foo() override;

Q99: What does final mean?

Prevent further overriding (method) or inheritance (class).

Q100: What is abstract class?

Class with at least one pure virtual function; cannot be instantiated.

Advanced Level

Q101: What is multiple inheritance?

Class inheriting from more than one base class.

Q102: What is diamond problem?

Ambiguity/duplication when two bases share common ancestor.

Q103: What is virtual inheritance?

Shares single base subobject in diamond hierarchies.

Q104: When use composition over inheritance?

Prefer composition for flexibility and weaker coupling unless true “is-a” relation exists.

Q105: What is pImpl idiom?

Pointer to implementation to hide details and reduce rebuild dependencies.

Q106: Benefits of pImpl?

ABI stability, faster incremental builds, encapsulation.

Q107: What is one-definition rule (ODR)?

Entities needing single consistent definition across program; violations are undefined behavior.

Q108: What is translation unit in C++?

Source file after preprocessing.

Q109: What is linkage (internal/external)?

Symbol visibility across translation units and linker scope.

Q110: What is static storage duration?

Object exists for entire program lifetime.

Q111: What is thread storage duration?

Per-thread object lifetime using thread_local.

Q112: What is dynamic storage duration?

Lifetime managed manually/indirectly via allocators/new/delete.

Q113: What is initialization order fiasco?

Cross-TU static initialization order is unspecified and can break dependencies.

Q114: How mitigate static init order issues?

Use function-local statics (construct-on-first-use) and dependency inversion.

Q115: Are function-local statics thread-safe?

Since C++11, initialization is thread-safe.

Q116: What is undefined behavior in C++?

Behavior not defined by standard; compiler may assume it never happens.

Q117: Common UB examples in C++?

Out-of-bounds access, use-after-free, data races, signed overflow, invalid casts.

Q118: What is implementation-defined behavior?

Compiler/platform must document chosen behavior (e.g., type sizes).

Q119: What is unspecified behavior?

Multiple valid outcomes without requirement to document which chosen each time.

Q120: What are strict aliasing rules?

Accessing object via incompatible type can trigger UB and optimizer miscompilations.

Q121: What is alignment?

Memory boundary requirements for objects/types.

Q122: What is alignas?

Specify stronger alignment requirements. 

struct alignas(64) CacheLineData { int x; };

Q123: What is constexpr function?

Function usable in compile-time contexts when given constant-expression inputs.

Q124: What is consteval?

Immediate function that must be evaluated at compile time.

Q125: What is constinit?

Ensures static/thread storage variable is constant-initialized.

Q126: What are concepts (C++20)?

Constraints on template parameters improving diagnostics and intent.

Q127: Why concepts help large codebases?

Earlier errors, clearer interfaces, better template readability.

Q128: What is requires clause?

Syntax to constrain templates with concept checks.

Q129: What is SFINAE?

Substitution failure is not an error; classic template constraint mechanism pre-concepts.

Q130: What is CRTP?

Curiously Recurring Template Pattern for static polymorphism.

Q131: Static vs dynamic polymorphism?

  • Static: compile-time (templates/CRTP), often zero-overhead
  • Dynamic: runtime virtual dispatch

Q132: What is type erasure?

Hide concrete type behind uniform interface (e.g., std::function, any-like wrappers).

Q133: What is std::any?

Type-safe container for single dynamically typed value.

Q134: What is std::variant?

Tagged union of known alternative types.

Q135: What is std::optional?

Wrapper representing optional presence/absence of value.

Q136: Why use optional instead of sentinel?

Explicit semantics and type safety.

Q137: What is std::span?

Non-owning view over contiguous sequence (pointer + length).

Q138: What is stringview?

Non-owning read-only view into character sequence.

Q139: Pitfall with stringview?

Dangling views if underlying string/buffer lifetime ends.

Q140: What is ranges library?

Composable lazy views and algorithms with cleaner pipeline style.

Q141: What is coroutine (C++20)?

Language support for suspend/resume computations (async generators/tasks).

Q142: Are coroutines threads?

No. They are control-flow abstraction; scheduling/execution model is library-defined.

Q143: What is move-only type?

Type supporting move but not copy (e.g., uniqueptr).

Q144: Why uniqueptr is move-only?

Enforces unique ownership.

Q145: What is sharedptr?

Reference-counted shared ownership smart pointer.

Q146: What is weakptr?

Non-owning observer of sharedptr-managed object; breaks cycles.

Q147: What is sharedptr cycle?

Mutual sharedptr references prevent destruction (leak-like lifetime).

Q148: How break sharedptr cycles?

Use std::weak_ptr for one direction in cyclic graph.

Q149: Why use makeunique/makeshared?

Exception safety and efficiency (single allocation for makeshared).

Q150: What is custom deleter?

Callable controlling cleanup behavior for smart pointers.

Expert Level

Q151: What is exception safety guarantee levels?

  • Basic: invariants preserved, no leaks
  • Strong: commit-or-rollback
  • Nothrow: operation never throws

Q152: What is copy-and-swap idiom?

Assignment via copy temp then swap for strong guarantee and reuse logic.

Q153: What is allocator in STL?

Policy object controlling memory allocation for containers.

Q154: Why custom allocators?

Specialized performance/memory behavior for domain workloads.

Q155: What is PMR (polymorphic memory resource)?

Runtime-pluggable allocation model in <memory_resource>.

Q156: What is cache-friendly data layout?

Organizing data for spatial/temporal locality to reduce cache misses.

Q157: What is false sharing in C++ multithreading?

Threads writing distinct vars on same cache line causing coherence contention.

Q158: How reduce false sharing?

Padding/alignment and per-thread sharding.

Q159: What is memory model in C++?

Formal rules for atomic operations, ordering, and visibility between threads.

Q160: What are atomic memory orders?

relaxed, consume (deprecated practice), acquire, release, acq_rel, seq_cst.

Q161: When use relaxed atomics?

For atomicity without synchronization ordering requirements.

Q162: Why is seqcst easiest to reason about?

Provides single global order for seqcst operations.

Q163: What is data race in C++?

Unsynchronized conflicting accesses where at least one is write and non-atomic => UB.

Q164: Is volatile for thread sync?

No. Use atomics/mutexes/condition variables.

Q165: What is mutex?

Primitive providing mutual exclusion for critical sections.

Q166: What is lockguard?

RAII wrapper that locks in ctor and unlocks in dtor.

Q167: What is uniquelock?

Flexible lock wrapper supporting defer, unlock/relock, conditionvariable integration.

Q168: What is deadlock?

Threads wait forever due to cyclic lock dependencies.

Q169: Deadlock prevention basics?

Consistent lock ordering, std::scopedlock, minimizing lock nesting.

Q170: What is conditionvariable?

Blocking wait/notify primitive for condition-based thread coordination.

Q171: Why wait with predicate?

Handles spurious wakeups safely. 

cv.wait(lock, [&]{ return ready; });

Q172: What is future/promise?

Mechanism for asynchronous result delivery and synchronization.

Q173: What is std::async caveat?

Launch policy matters; may run deferred unless std::launch::async requested.

Q174: What is lock-free programming challenge?

Correctness complexity (ABA, memory reclamation, ordering bugs).

Q175: What is ABA problem?

Compare-exchange sees same value again though state changed in between.

Q176: What is hazard pointer/epoch reclamation?

Techniques for safe memory reclamation in lock-free structures.

Q177: What is modules in C++20?

Alternative to textual includes for faster, cleaner dependency boundaries.

Q178: Benefits of modules over headers?

Faster builds, better isolation, reduced macro leakage.

Q179: What is build system’s role for C++?

Manage compilation units, dependencies, flags, linking, test orchestration.

Q180: Why compile with high warnings?

Catch bugs early and enforce code quality consistency.

Q181: Useful warning set baseline?

-Wall -Wextra -Wpedantic plus project-specific additions.

Q182: Why test with multiple compilers?

Different diagnostics and optimizer behaviors reveal different bugs.

Q183: What is sanitizer suite for C++?

  • ASan: memory errors
  • UBSan: undefined behavior
  • TSan: data races
  • MSan: uninitialized reads (toolchain support dependent)

Q184: Why use fuzzing in C++?

Finds edge-case crashes/logic flaws in parser and API boundaries.

Q185: What is benchmark trap?

Microbenchmarks may mislead if not representative/well-isolated.

Q186: What is profile-guided optimization?

Use runtime profiles to optimize branch/layout/inlining decisions.

Q187: What is link-time optimization?

Whole-program optimization during link for better inlining/devirtualization.

Q188: What is devirtualization?

Compiler optimization replacing virtual dispatch with direct call when type known.

Q189: What is ABI compatibility concern in libraries?

Changing class layout/signatures/symbols can break binary clients.

Q190: How maintain ABI stability?

pImpl, symbol versioning, careful exported interfaces, semantic versioning discipline.

Q191: Why avoid exposing STL types in stable ABI boundaries?

Implementation and ABI may vary across compilers/standard library versions.

Q192: What is strong typedef pattern?

Distinct wrapper types to avoid mixing semantically different primitives.

Q193: What is value semantics?

Objects behave like values (copyable/movable, independent ownership).

Q194: Why prefer value semantics by default?

Simpler reasoning, fewer aliasing/lifetime bugs.

Q195: What is “zero-cost abstraction” in C++?

High-level abstraction that compiles to cost comparable to hand-written low-level code.

Q196: What is guideline for modern C++ API design?

Clear ownership, const-correctness, minimal raw pointers, expressive types, narrow interfaces.

Q197: Should you expose raw pointers in interfaces?

Use for non-owning nullable observation when semantically correct; document lifetime rules.

Q198: What distinguishes expert C++ code from average code?

Correctness under complexity, explicit lifetimes/ownership, measurable performance, maintainable abstractions.

Q199: What is practical migration path from legacy C++?

Introduce RAII/smart pointers, modern containers, safer APIs, tests/sanitizers, incremental refactors.

Q200: What is expert workflow for C++ development?

  1. Define contracts and invariants
  2. Prefer simple/clear design first
  3. Enforce warnings/static analysis
  4. Test + fuzz + sanitizers
  5. Profile before optimizing
  6. Review for lifetime/concurrency/UB risks
  7. Document API and performance assumptions
